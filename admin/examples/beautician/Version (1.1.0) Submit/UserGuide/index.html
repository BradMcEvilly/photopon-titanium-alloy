How to replace another icon? {Replace another font icon}

Replace Icon

If you wish to replace another font icon, here is how to do.

1. Available Fonts Icon List

There is “FontIcon” folder inside our main files. When you open demo.html file, you will see something like this.

2. Copy Font Icon Name

You need to copy that icon name to change another icon.

3. Example

If you want to change “Profile” icon( From Side Menu) to another icon, you need to change at here.

var profileIcon = fontIconLoader.getIcon("panacea","profile",35,{color:Alloy.CFG.Colors.SideMenuIconColor});

profile is icon name, so you need to replace that icon name which you copy at step 2.

Let’s say if you copy “profile2” at step 2. It will be like this.

var profileIcon = fontIconLoader.getIcon("panacea","profile2”,35,{color:Alloy.CFG.Colors.SideMenuIconColor});

Note : If you new icon is not refresh, please uninstall and install your app.

How to add new icon? {Add new font icon}

New Icon

Actually, you can get a lot of icons at our “panacea” font. But if you want to add new icon, here is the step.

1. Import "selection.json"

There is “selection.json” at “FontIcon” folder. You can import *selection.json* back to the IcoMoon app using the *Import Icons* button (or via Main Menu > Manage Projects) to retrieve your icon selection. This is font icon generating website https://icomoon.io/

2. Playing at icomoon.io

Assume that you already import your icons at icomoon.io. Then you need to download that new font icon. Please click to “Download” button at bottom of page.

3. selection.json to panacea.json

After download and unzip, you will see like our “FontIcon”. You copy that “selection.json” file into Ti project folder (app\assets\fontmaps). Then you replace file name as “panacea.json”.

4. panacea.ttf

You need to copy “panacea.ttf” from “fonts” folder into “app\assets\fonts”. If already have, you can override it.

5. Checking

You can make sure you imported font inside or not. So, please open “panacea.json” from “app\assets\fontmaps” You can search by typing your icon name.

6. New Font Icon

Assume that everything is going well and you can see your new icon also inside. Then you just need to replace at that line.

var profileIcon = fontIconLoader.getIcon("panacea","oldicon”,35,{color:Alloy.CFG.Colors.SideMenuIconColor});

var profileIcon = fontIconLoader.getIcon("panacea”,”newicon”,35,{color:Alloy.CFG.Colors.SideMenuIconColor});

User Role & Access {About different user role and access right?}

There are Three types of user roles. They are

  • Administrator
    Full access for system such as add, edit, delete, publish and ban.
  • Manager
    Partial access for system such as add, edit, delete and publish.
  • User
    Basic access for system such as add and edit.

Module Development{How to customize?}

If you want more than our default module, you may need to customize on it. so we would like to share how you can extend from your system.

Module Access

For Eg; You can access module access.

1. Create Module Data

Please add your module name(Controller Name) and module description(Description at Sidebar Menu) at modules table.

2. Set Module Permission

Please create controller with module name under beautician/application/controllers folder.

require permission.php
extends permission class
call parent::__construct() at constructor with module name as a parameter
for add function, call $this->check_access('add');
for edit function, call $this->check_access('edit');
for delete function, call $this->check_access('delete');
for publish function, call $this->check_access('publish');
for ban function, call $this->check_access('ban');
for view function, you can use template system like $this->load->view('template',$data);
Create New Module

For Eg; Let's create new book module.

Name Description
modules Please insert at Module Table from Database.
module_name = 'books' module_description = 'Books'
Controllers Please create Books Controller(books.php) under beautician/application/controllers folder. 

require_once('permission.php');
class Books extends permission{
 function __construct(){
     parent::__construct('books');
 }
  
 function add(){
     $this->check_access('add');
     //you code here
 }
  
 function index(){
     //parameter must be array and key must be 'content'
     $this->load->view('template',
     array('content'=>'Hello, World!'));
 }

 function edit(){
     $this->check_access('edit');
     //you code here
 }
}
});
Views You could create all your view files under "application/views/books/" folder
Models You could create all your view files under "application/models" folder
Permission Please give Permission yourself for that module.To access the module, go to the backend. Edit yourself at System Users Module and give permission for that module.
autoload.php Please make sure to load your model at config\autoload.php
